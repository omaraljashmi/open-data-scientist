# Data Insight Studio v1.0.0-rc.1

This first release candidate packages the complete local-first Data Insight Studio workflow for public testing: profile a file, review cleaning recommendations, build guided or custom dashboards, create Visual SQL, and inspect the query with SQL Coach.

## Release-candidate highlights

- Try the complete product with one click using the bundled sample dataset.
- Understand the hosted-demo privacy boundary before uploading anything.
- Get clear failures for unsupported, empty, oversized, malformed, binary, encrypted, or unexpectedly expanded files.
- Reproduce the verified environment with `requirements.lock`.
- Run Data Insight Studio locally in Python or as a health-checked Docker container.
- Verify the app through Python 3.11/3.12 CI, Streamlit UI smoke tests, calculation tests, safety tests, and a deterministic performance gate.

## Deliberate boundaries

- The public-demo envelope is 25 MB, 250,000 rows, and 500 columns.
- The public demo is hosted; sensitive data belongs in a local installation.
- Data Insight Studio is a single-file, single-user analytical workflow, not a collaborative BI server.
- SQL optimization remains rule-based and DuckDB-aware; no paid API or external AI model is required.
- OR filter groups, multi-file joins, and optional local-model support remain future milestones.

## Test this candidate

1. Click **Try sample dataset** on the landing page.
2. Review quality evidence and a proposed cleaning action.
3. Check a Dashboard Studio card's calculation table.
4. Build a Visual SQL summary and send it to SQL Coach.
5. Download a layout JSON or offline dashboard HTML.

Please report bugs without attaching private data. See `PRIVACY.md` and `SECURITY.md` before testing with your own files.

## Local verification record

- 80 automated tests across UI smoke, loading guardrails, profiling, cleaning, dashboards, Visual SQL, SQL Coach, performance, and release metadata
- clean Python 3.12 installation of all 50 locked packages with no dependency conflicts
- 100,000-row deterministic benchmark completed in 1.776 seconds in the verification environment
- Docker build and health check configured as a required CI job because the local verification environment has no container engine
