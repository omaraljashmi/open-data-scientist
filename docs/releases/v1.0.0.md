# Open Data Scientist v1.0.0

Open Data Scientist 1.0 is the first stable local-first workflow for turning a CSV or Excel file into an explainable quality profile, reviewable cleaning plan, auditable dashboard, safe visual query, and SQL optimization evidence without a paid API.

## Stable highlights

- Complete one-click product walkthrough using a bundled synthetic dataset.
- Deterministic profiling, cleaning, dashboards, Visual SQL, and SQL Coach outputs.
- Privacy-aware hosted demo plus documented local-installation boundary.
- Guardrails for upload size, dataset shape, malformed content, and XLSX expansion.
- Exact release dependency lock, Docker packaging, and Python 3.11/3.12 validation.
- Multi-format stable-release matrix covering common CSV variations, XLSX, and legacy XLS.
- Privacy-safe public issue forms and contribution checklist.

## Validation record

- Seven accepted synthetic files complete the full workflow.
- Nine invalid or unsafe files fail with an expected user-facing error family.
- Single-column CSV handling no longer invents a delimiter from data values.
- Blank and duplicate CSV headers are rejected before pandas can silently rename them.
- Locked local Python 3.11 and Python 3.12 validation environments each pass all 86 automated tests with compatible dependencies.
- The deterministic 100,000-row benchmark completes in 0.618 seconds in the clean Python 3.12 stable-preparation environment.
- GitHub Actions run `29389780191` passes the complete Python 3.11/3.12 matrix, Docker image build, and Streamlit container-health smoke test.
- The public Streamlit deployment completes the bundled sample walkthrough without a rendered exception.

## Deliberate boundaries

- The default envelope remains 25 MB, 250,000 rows, and 500 columns.
- The public demo is hosted; confidential or regulated data belongs in a reviewed local deployment.
- ODS analyzes one uploaded file for one interactive user; it is not a collaborative BI server.
- SQL guidance is deterministic and DuckDB-aware; business correctness still requires domain review.
- Optional local-model support, OR filter groups, and multi-file joins remain future milestones.

## Upgrade from the release candidate

After the stable tag is published:

```bash
git fetch --tags
git checkout v1.0.0
python -m pip install -r requirements.lock
python -m streamlit run app.py
```

Review `PRIVACY.md`, `SECURITY.md`, and `docs/VALIDATION.md` before using your own data.
